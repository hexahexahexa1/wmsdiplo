# Исправление: Улучшенная компоновка панели фильтров

## Проблема
После увеличения ширины кнопки "Очистить" до 150px, кнопки не помещались в одной строке с фильтрами, что привело к обрезанию текста на кнопке "Поиск".

## Решение
Разделена панель фильтров на **две строки** для лучшей читаемости и компоновки:

### Новая структура:

**Строка 1 (Фильтры)**:
- Код SKU (150px)
- Код ячейки (150px)
- Баркод паллеты (150px)
- ID прихода (150px)
- Статус (150px)

**Строка 2 (Кнопки)**:
- Поиск (120px)
- Очистить (120px)

### Преимущества нового дизайна:
✅ Все элементы полностью видны  
✅ Улучшена читаемость интерфейса  
✅ Логическое разделение: фильтры отдельно, действия отдельно  
✅ Кнопки теперь выровнены по левому краю для консистентности  
✅ Все поля фильтров имеют одинаковую ширину (150px) для визуальной гармонии

### Изменения в коде:
**Файл**: `desktop-client/src/main/java/com/wmsdipl/desktop/DesktopClientApplication.java`

1. Увеличена ширина полей фильтров для единообразия:
   - `receiptIdFilter`: 120px → 150px
   - `statusFilter`: 130px → 150px

2. Создана отдельная строка для кнопок:
   ```java
   HBox filtersRow = new HBox(10, skuCodeFilter, locationCodeFilter, 
                              palletBarcodeFilter, receiptIdFilter, statusFilter);
   
   HBox buttonsRow = new HBox(10, searchBtn, clearBtn);
   ```

3. Обновлен layout для включения обеих строк:
   ```java
   VBox layout = new VBox(12, header, filtersRow, buttonsRow, stockTable, paginationBox);
   ```

4. Кнопки вернулись к стандартной ширине 120px (достаточно для текста)

## Применение изменений

Перезапустите desktop-client:

```bash
# Остановите текущий процесс (если запущен)
# Затем:
gradle :desktop-client:run
```
